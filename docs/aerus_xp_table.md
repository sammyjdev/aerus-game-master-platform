## 1. Princípio de Progressão

O Aerum vai até o nível 100. A curva é **híbrida**:
- **Níveis 1–50:** progressão linear — crescimento constante, mundo acessível
- **Níveis 51–100:** progressão exponencial — cada nível representa uma maestria crescentemente rara

O level up é uma **sensação**, não um evento. Quando o personagem atinge XP suficiente, algo muda internamente — uma clareza nova, reflexos mais afiados, uma compreensão que não existia antes. Nada é desbloqueado de imediato. Novas habilidades exigem **treinamento** para se manifestar.

## 3. Fontes de XP

### 3.1 Feitos Dignos

O principal gatilho de XP. Um feito digno é qualquer ação que resolva um obstáculo real, salve alguém, faça a história avançar, ou exija sacrifício pessoal.

| Escala do Feito | XP |
|---|---|
| Menor (ajudar NPC, descobrir pista) | 25–50 |
| Significativo (completar objetivo de missão, revelar segredo importante) | 100–300 |
| Maior (prevenir catástrofe, sacrifício narrativo de peso) | 500–1.000 |
| Marco de campanha (arc completada, evento irreversível) | 2.000–5.000 |

### 3.2 Sucessos em Combate

XP é concedido por **resolver** o combate — eliminar inimigos, incapacitar, ou atingir o objetivo da cena de combate.

| Tier do inimigo | XP base por combate resolvido |
|---|---|
| Tier 1 | 10–25 |
| Tier 2 | 50–100 |
| Tier 3 | 150–350 |
| Tier 4 | 500–1.200 |
| Tier 5 | 2.000–5.000 |
| Boss (qualquer tier) | XP do tier × 3 |

### 3.3 Lutar Contra Oponentes Mais Fortes

Confrontar inimigos acima do nível do grupo, mesmo sem vencer, gera XP pelo esforço e aprendizado:

| Diferença de tier | Multiplicador sobre XP base |
|---|---|
| Mesmo tier | ×1 |
| 1 tier acima | ×1,5 |
| 2 tiers acima | ×2 |
| 3+ tiers acima | ×3 (cap) |

**Treino e sparring:** confrontar um oponente mais forte em contexto controlado (treino com mentor, sparring em arena) concede **30% do XP normal** do inimigo.

## 4. Level Up em Sessão

Quando um personagem acumula XP suficiente para avançar de nível durante uma cena ativa:

1. **O GM narra o momento** — não interrompe a cena, mas sinaliza o instante:
   > *"No meio do movimento, algo encaixa. Como se um músculo que [personagem] não sabia que existia acordasse. A ação continua — mas algo mudou."*

2. **Nada é desbloqueado imediatamente.** O nível é registrado, mas novas habilidades, bônus de atributo e poderes requerem **treinamento** entre sessões ou durante descanso longo.

3. **Dois personagens evoluindo ao mesmo tempo:** o GM narra separadamente para cada um na mesma resposta, em blocos — primeiro o de maior iniciativa, depois o segundo.

## 5. Mutações Formais (Níveis 25 / 50 / 75 / 100)

Nos níveis 25, 50, 75 e 100, o sistema `behavior_trajectory.py` avalia o histórico de ações do personagem e **sugere 3 caminhos de mutação** com base no padrão de comportamento observado.

- **O jogador não sabe que está sendo observado.**
- As 3 opções apresentadas pelo GM são contextualizadas narrativamente — não como escolha de menu, mas como possibilidades que emergiram da trajetória do personagem.
- **O jogador escolhe entre as 3.**
- A mutação entra em vigor após um período de **treinamento narrativo** (definido pelo GM — tipicamente uma cena de treino, meditação, ou evento de transformação).

### 5.1 Protocolo Narrativo da Mutação

> *"[Personagem] sente o ponto de inflexão — não como decisão, mas como reconhecimento. Três caminhos se tornaram nítidos. Qual deles já estava sendo percorrido?"*

O GM apresenta as 3 opções de forma descritiva, sem termos mecânicos — a mutação é uma identidade emergindo, não uma habilidade sendo comprada.

**Se o jogador rejeitar todas as 3:** incomum, mas possível. O GM registra e o sistema recalcula baseado nos próximos 5 níveis de comportamento. A mutação é adiada — não cancelada.

**Mutação durante combate ativo:** não acontece. O sistema aguarda o fim da cena.

## 6. Milestones Passivos (a cada 25 pontos de XP base)

Conforme `campaign.yaml: passive_milestone_every_points: 25`, o sistema verifica milestones a cada 25 pontos de XP concedidos.

Milestones passivos **não interrompem a cena**. São verificações silenciosas de threshold (atributo atingindo cap, skill desbloqueada por uso repetido, etc.). O GM pode narrar o efeito na próxima pausa natural da cena.
